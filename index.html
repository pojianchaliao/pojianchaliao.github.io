开发遇到的坑

JAVA web

  • ORM object relation 对象关系映射 ，用于从数据库中取数据并封装到对象的语句
  • static 静态代码块，只执行一次，一个项目工厂只创建一次 （单例模式）
  • java空指针，常见原因:字符串没有初始化，类没有初始化（在trycatch中可能是catch出了错误没有执行初始化操作）没有判断为空。 31:40
  • date 有sqldate和 util.date区别
  • $.get()将表单通过url传参给servlet如果form是hidden值全为null
  • 更新 若其中还有href=’ ‘ updateUser只会显示一瞬间 然后被url提交因为url为空所以回到原界面
  • IOC 里给类中的属性注入值可以通过set generate Autowired（自动装配（容器会查找和属性类型相匹配的bean组件注入当存在两个以上相同类型的组件时用@Qualifier区别）） 注入
  • 注解未命名是默认是类的首字母小写的单词
  • JS文件被浏览器缓存造成语句无法更新的问题（可以加时间戳或者加版本时间）
  • @RequestParam 中的入参的值的名字如果和url对应的对象的名字不一样会报400的错误
  • 多表查询时，两个表的列名相同时mybati会出错，可以给其中一个类取别名
  • JS验证时，如果表单里的提交按钮是submit 那么无论js判断是否通过都会提交表单
  • 编写了mvc的配置类 如果是的话那就继承WebMvcConfigurerAdapter，不要继承WebMvcConfigurerationSupport,继承后者表明完全接管springMvc,会导致springBoot的默认设置都丢弃掉。
  • 实体类和数据库的列名不同时通过类进行增删改查需要Param
  • 在一个表格中的<select><option>选择点击事件不知道用什么方式实现（如果是在select使用onchange（js）不知为何只能获取到第一行的状态，如果是用option的id也只能获取第一行）（后改用ul li 表达下拉框）
  • 2020/6/24 表单提交时，同类型的select提交没问题，有一个select的值变成了null，后端为null，前端name属性已经设置了，并且js动态加载时已经设置了select下的各个option的id和value
  • 底层用update 为什么会出现新的记录 （当没有对应查询条件即id或其他条件匹配时）
  • mapper层如果在insert语句里使用id自增，用来插入数据时返回对应的id ，如果没有用对应实体类当dao的参数而是用实体类里的具体属性做dao的参数，那么返回插入id的效果会失效。
  • 实体类整数用int时，前端如果传来的值是null ，会报错无法将null转换为int 所以需要用integer

Android

  • pojo中的数据名最好和gson中的数据名一样，不然需要加注解 数据库连接判断isnull 在isopen前，若是null状态，不存在isopen方法，会崩溃
  • alt+ctrl+v 自动补全前面的对象 alt +enter 修改错误 alt+insert 生成构造函数 ctrl+alt+l 代码格式化 ctrl+/ 注释反注释
  • ctrl+alt+T 生成trycatch if 等条件语句 泛型数组 适配器 main里面调用方法返回City等类型的数据，再把city数据通过适配器绑定到listview里
  • task9 发生线程里面要改UI需要activity asynctask 的继承类需要其他参数时可以用构造函数调用使用构造函数的类传进来的参数
  • shift+end 定位行尾
  • edittext 如果需要动态的取到它的值需要把find..ById 放到监听事件里面
  • arraylist在初始化的时候如果直接赋值为null 那么后面就不能用add 插入值

Linux 和Windows区别

  • ls 和 dir vim 和 notepad
